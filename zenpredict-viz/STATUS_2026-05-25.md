  - The 228 / 300 / 372 zensim schemas, each with per-feature

    macro body in `zenanalyze/src/feature.rs`.
- UI: `feature_layout.js` async-loads the catalog at boot;
  importance panel adds a feature-search box (substring match
  on label / block / feature name / math summary) that dims
  non-matching cells; tooltips show math summary + source ref.
- `build.sh` runs the catalog binary after the WASM build so a
  single `./build.sh release` produces both artifacts.
- `training_stats` block reserved in schema but not yet populated
  — wiring a parquet reader is a follow-up.

### Track C — Stage 2 P1 weights + calibration panels (commit `e9665a7f36b9`)

- `Weights` tab (`web/panels/weights.js`): per-layer selector,
  Canvas-based heatmap (symmetric diverging palette, hover for
  `W[input_label, h_idx]` with semantic labels on layer 0),
  bias/scale side strips, weight-value histogram.
- `Calibration` tab (`web/panels/calibration.js` +
  `web/calibration_decoders.js`): JS-side TLV decoders mirror the
  parse_* helpers in `zensim/src/metric.rs`. Renders only stages
  present on the bake (absent stages show a `(not present)`
  placeholder so the ordered pipeline is always visible):
  - `tanh_output_head` → `100·σ(x/scale)` line plot
  - `output_calibration_spline` → PCHIP curve with knot dots + knot
    table
  - `per_codec_calibration` → table + overlay line plot vs identity
  - `per_sample_alpha_head` → architecture diagram + rank_w /
    w_alpha histograms
- `Forward` panel waterfall extended to apply the calibration
  stages in pipeline order (tanh-pin → spline → per-codec affine).
  Final score line distinguishes raw vs calibrated; per-codec
  table shows the score for the current input under each codec
  affine.

### Track D — Bake compare + feature attribution (commit `<this-session>`)

- `Compare` tab (`web/panels/compare.js`): load 2–4 bakes with
  matching schemas (n_inputs / n_outputs / per-layer dims).
  Baseline is the first; later bakes show:
  - Scaler shift heatmap (Δ mean, Δ scale).
  - L0 importance reshuffle heatmap + top-20 movers table.
  - Per-layer weight RMS + L2 norm of the weight diff vs baseline.
  - Calibration curve overlay for `tanh_output_head` and
    `output_calibration_spline` when both bakes carry the stage.
- `Attribution` tab (`web/panels/attribution.js`): MVP first-layer
  attribution: `attribution[i] = standardized[i] · Σ_h |W₀[i, h]|`.
  Surfaces top positive + top negative contributors with semantic
  labels. Real SHAP / IG would integrate downstream gradients;
  the MVP is one matmul pass and still strongly indicative for
  "why did this image score X?" diagnosis.

  `forward_with_taps` WASM bridge.

  and emits p5/p25/median/p75/p95/max per feature. Probably a
  Python pre-step rather than Rust (arrow-rs/parquet is heavy and
  this is a one-shot per training-corpus revision).
- **Calibration stage 4 — per_sample_alpha_head curve view** —
  currently shown as an architecture diagram; a forward-pass
  visualisation that lets the user vary the hidden vector and
  watch the alpha gate is a richer mode. Needs a "given features,
  show me the alpha mix" panel; deferred.
- **Compare panel weight-diff heatmaps** — current MVP shows RMS
  + L2 norm; a per-layer diff heatmap (B - A) would help diagnose
  where the bakes diverge most. Same code as the Weights heatmap
  with subtracted values; ~30 min of work.
- **Netron screenshot** — `docs/netron_v11.png` referenced in the
  brief; not generated (Netron is a GUI; the netron.app URL works
  as the canonical "see this rendered" reference).
